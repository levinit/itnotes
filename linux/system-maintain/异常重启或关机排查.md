# Linux 异常重启 / 异常关机排查



## 一、目标

当出现以下情况时使用：

* 系统突然 reboot
* 无法解释的宕机
* 没有正常 shutdown 记录
* 计算节点异常消失 / 被调度系统踢掉



## 二、排查思路总览

```
是否人为操作？
    ↓
是否正常 shutdown？
    ↓
是否内核问题（OOM / panic）？
    ↓
是否硬件/BMC触发？
    ↓
是否电源/机房问题？
```



## 三、系统层面排查

### 3.1 查看重启时间

```bash
who -b
uptime
last reboot
```



### 3.2 是否正常 shutdown

```bash
last -x | grep shutdown
```

#### 判断

| 结果         | 说明       |
| - | -- |
| 有 shutdown | 正常关机     |
| 无 shutdown | 异常重启（重点） |



### 3.3 是否人为操作

```bash
grep -iE "reboot|shutdown" /var/log/secure
ausearch -x reboot
ausearch -x shutdown
```

#### 判断

| 情况    | 说明   |
| -- | - |
| 有用户记录 | 人为操作 |
| 无记录   | 非人为  |



### 3.4 systemd 日志（关键）

```bash
journalctl -b -1
```

#### 常见关键词

| 关键词       | 含义    |
|  --- | -- |
| Reboot    | 正常重启  |
| Power-Off | 正常关机  |
| panic     | 内核崩溃  |
| watchdog  | 看门狗重启 |

如果出现：

```
no persistent journal was found
```

说明日志丢失，需要修复（见后文）



### 3.5 内核日志

```bash
journalctl -k -b -1
```

或（当前运行）：

```bash
dmesg
```



### 3.6 OOM 检查

```bash
dmesg | grep -i oom
journalctl -k | grep -i "out of memory"/m
```



### 3.7 panic / crash

```bash
grep -i panic /var/log/messages
```



## 四、硬件 / BMC 层面

### 4.1 查看 SEL

SEL是*System Event Log，*可*查看各种硬件告警*

```bash
ipmitool sel list
```

输出示例，一条内存条ECC错误：

> 5a | 01/01/2046 | 12:34:56 | Memory #0x02 | Uncorrectable ECC (DIMMA4) | Asserted

其中第1列是日志编号，可以用以下命令查询详情：
```shell
ipmitool sel get 0x5a  #需要加上0x前缀
```



### 4.2 常见 SEL 事件解释

| 事件            | 含义          |
| - | -- |
| Power Off     | 掉电 / 被断电    |
| Power On      | 上电          |
| Power Cycle   | 电源重启        |
| System Reset  | 硬件触发 reset  |
| Watchdog      | watchdog 重启 |
| Temperature   | 温度问题        |
| Fan Fail      | 风扇故障        |
| PSU Failure   | 电源模块异常      |
| Memory        | 内存问题        |
| Processor     | CPU问题         |

事件名字不一定和真实`impitool sel list`输出一致，但是一般有类似关键词。输出信息的最后一列有详细说明，用于明确排查到详细信息。



### 4.3 判断逻辑

| SEL 情况               | 结论             |
| -- | -- |
| Power Off + Power On | 掉电             |
| Reset                | BMC / watchdog |
| 无记录                  | 可能 OS crash    |



## 五、常见异常类型归类

### 5.1 掉电（最常见）

特征：

* 无 shutdown
* 无系统日志
* SEL 有 Power Off



### 5.2 watchdog 重启

特征：

* SEL 有 watchdog
* 系统无明显日志



### 5.3 kernel panic

特征：

* 日志有 panic
* 可能自动重启



### 5.4 OOM

特征：

* Out of memory
* 进程被杀

注意：OOM **不会直接 reboot**



### 5.5 人为操作

特征：

* secure 有记录
* audit 有记录



### 5.6 BMC远程操作

来源：

* IPMI
* iDRAC / iLO



## 六、必须开启的日志

### 7.1 开启 persistent journal

```bash
mkdir -p /var/log/journal
systemctl restart systemd-journald
```



### 7.2 开启 kdump（抓 panic）

```bash
yum install kexec-tools
systemctl enable kdump
systemctl start kdump
```



### 7.3 audit 记录 reboot

```bash
auditctl -a exit,always -F arch=b64 -S reboot -k reboot_log
```



### 7.4 集中日志（推荐）

* rsyslog server
* ELK / Loki



## 七、标准排查命令集合（直接用）

```bash
# 时间
who -b
last reboot

# shutdown
last -x | grep shutdown

# 用户行为
grep -i reboot /var/log/secure

# journal
journalctl -b -1
journalctl -k -b -1

# OOM
dmesg | grep -i oom

# 硬件
ipmitool sel list
```
